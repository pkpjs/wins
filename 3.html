HTTP 접속 오류 코드

아래는 HTTP 상태 코드와 그 의미를 설명한 표입니다.

1xx - 정보 응답 (Informational)

코드 상태 설명
100 Continue: 요청이 부분적으로 수신되었으며, 클라이언트가 요청을 계속 진행해야 함.
101 Switching Protocols: 서버가 클라이언트의 프로토콜 전환 요청을 승인했음을 알림.

2xx - 성공적인 응답 (Successful)

코드 상태 설명
200 OK: 요청이 성공적으로 처리됨.
201 Created: 요청이 성공적으로 처리되어 새로운 자원이 생성됨.
204 No Content: 요청은 성공했지만 응답할 콘텐츠가 없음.

3xx - 리디렉션 (Redirection)

코드 상태 설명
301 Moved Permanently: 요청한 자원이 영구적으로 다른 위치로 이동됨.
302 Found: 요청한 자원이 임시로 다른 위치에 있음.
304 Not Modified: 자원이 변경되지 않았으며, 캐시된 버전을 사용할 수 있음.

4xx - 클라이언트 오류 (Client Errors)

코드 상태 설명
400 Bad Request: 서버가 요청을 이해할 수 없거나 잘못된 요청이 들어왔음.
401 Unauthorized: 인증이 필요함. 사용자 인증이 실패했을 때 발생.
403 Forbidden: 요청이 서버에 의해 거부됨. 권한이 없을 경우 발생.
404 Not Found: 요청한 자원을 찾을 수 없음.
405 Method Not Allowed: 요청한 HTTP 메서드가 허용되지 않음.
408 Request Timeout: 클라이언트의 요청이 시간 내에 완료되지 않음.

5xx - 서버 오류 (Server Errors)

코드 상태 설명
500 Internal Server Error: 서버 내부에서 오류가 발생하여 요청을 처리할 수 없음.
502 Bad Gateway: 서버가 게이트웨이 또는 프록시로서 작동 중, 업스트림 서버에서 오류 발생.
503 Service Unavailable: 서버가 일시적으로 사용 불가능. 과부하 또는 서버 유지 관리 등.
504 Gateway Timeout: 서버가 게이트웨이로서 작업을 진행하는 동안 시간이 초과됨.
